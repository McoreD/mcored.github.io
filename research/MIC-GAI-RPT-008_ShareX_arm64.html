Technology Research & Justification

Analysis of hardware acceleration and library choices for the ARM64 ecosystem.

SkiaSharp vs. System.Drawing

Legacy System.Drawing relies on GDI+, which performs poorly on non-x86 architectures.

SkiaSharp: Hardware accelerated via GPU (OpenGL/Vulkan/Metal). Native ARM64 bindings available.

GDI+: CPU-bound software rendering on ARM64.

Video Encoding (FFmpeg)

Screen recording requires efficient encoding to prevent CPU throttling on mobile chips.

Native FFmpeg: Requires compiling generic builds for aarch64.

Media Foundation: Access hardware encoders (Qualcomm/Adreno) directly via Windows SDK.

Capture APIs

Method of capturing screen pixels determines frame rate and battery life.

Windows.Graphics.Capture: Modern, GPU-efficient, zero-copy capabilities.

DXGI Desktop Dup: Robust but varying support on mobile drivers.
